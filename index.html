2. Paragraph Style in markdown github flavour

            সাধারণত যেভাবে p tag এর মধ্যে আমরা paragraph লিখে থাকি
            markdown এ ও একইভাবে লেখা যায়। এতে কোনো ভিন্নতা নেই। তবে
            লেখার align right/center করার নিয়মটি হলো...
            <p align="center">This sentence shows in center</p>
            লেখাকে এক একটি লাইনে বিন্যস্ত করার জন্য enter দিয়ে পরবর্তী লাইনে গিয়ে লিখলেই 
            তা এক একটি লাইনে বিন্যস্ত হয়ে যাবে।

3. Emphasis Style in markdown github flavour

            লেখাকে bold করার জন্য এখানে ব্যবহার করা হয় যথাক্রমে...
            **Bold** (পূর্বে এবং পরে পর পর দুটি করে স্টার চিহ্ন দিতে হবে)
            __bold__ (পূর্বে এবং পরে পর পর দুটি আন্ডারস্কোর চিহ্ন দিতে হবে) 
            লেখাকে italic করার জন্য এখানে ব্যবহার করা হয় যথাক্রমে...
            *italic* (পূর্বে এবং পরে একটি স্টার চিহ্ন দিতে হবে)
            _italic_ (পূর্বে এবং পরে একটি আন্ডারস্কোর চিহ্ন দিতে হবে)
            লেখাকে একই সাথে bold এবং italic করা হয় যথাক্রমে...
            **_bold&italic_** 
            (পূর্বে পর পর দুটি স্টার ও একটি আন্ডারস্কোর চিহ্ন এবং 
            পরে একটি আন্ডারস্কোর ও পর পর দুটি স্টার চিহ্ন দিতে হবে)
            __*bold&italic*__
            (পূর্বে পর পর দুটি আন্ডারস্কোর ও একটি স্টার চিহ্ন এবং 
            পরে একটি স্টার ও পর পর দুটি আন্ডারস্কোর চিহ্ন দিতে হবে)
            লেখার মধ্যে দিয়ে লাইন টানা অর্থাৎ strikethrough করা হয় যথাক্রমে...
            ~~strikethrough~~(পূর্বে এবং পরে পর পর দুটি টিল্ডার চিহ্ন দিতে হবে)

4. Blockquote Style in markdown github flavour

            কোনো লেখাকে blockquote আকারে দেখানোর জন্য > চিহ্ন দিতে হবে।
            যদি আরো কোনো sub blockquote দেখাতে হয় সেক্ষেত্রে >> চিহ্ন দিতে হবে।

5. Link Style in markdown github flavour

            লিংক প্রদর্শন করার পদ্ধতি সমূহ যথাক্রমে...
            [Google](https://google.com) 
            (এতে করে শুধু Google লেখাটি লিংকে পরিণত হবে)
            [Google](https://google.com "search engine")
            (এতে করে Google লেখায় hover করলে টাইটেল দেখাবে search engine)
            
            (এতে করে সরাসরি লিংক হয়ে যাবে)
            [google][1]
            [google][2]
            [1]: <https://google.com>
            [2]: <https://google.com> "search engine" (টাইটেল দেখানোর জন্য)
            (কোনো paragraph এর মধ্যে লিংক জাতীয় word এর ব্যবহার বেশি হলে সেক্ষেত্রে
            এভাবে [1][2] সিরিয়াল আকারে লিখে পরবর্তীতে নিচে সেই সকল সিরিয়ালের লিংক তৈরি
            করা যায়। এতে করে কোড হিজিবিজি মনে হয় না এবং রিডেবল হয়।)

6. List Style in markdown github flavour

            দুই ধরণের লিস্ট আমরা দেখে থাকি order &  unorder list
            এই order & unorder list লেখার পদ্ধতি যথাক্রমে... 
            1. Item one (যেকোনো একটি সংখ্যা,ডট এবং একটি স্পেস)
            1. Item two 
                1. sub item one (ট্যাব অথবা ৪টি অক্ষরের সমপরিমাণ খালি রাখা)
                1. sub item two
            - unorder item one (একটি ড্যাস চিহ্ন তারপর স্পেস) 
            - unorder item two 
                - unorder sub item one (ট্যাব অথবা ৪টি অক্ষরের সমপরিমাণ স্পেস)
                - unorder sub item two

7. Checkbox Style in markdown github flavour

            দুই ধরণের checkbox তৈরি করার নিয়মাবলি যথাক্রমে... 
            - [ ] unchecked (একটি ডেস এবং তৃতীয় বন্ধনীর মধ্যে একটি স্পেস)
            - [x] checked (একটি ডেস এবং তৃতীয় বন্ধনীর মধ্যে একটি এক্স)

8. Collapsible Style in markdown and github flavour

            কোন paragraph/heading/link প্রভৃতিকে collapsible করা হয় যথাক্রমে... 
            <details> (এতে করে collapsible arrow তৈরি হবে)
                <summary>Click to see what is inside</summary>
                (এতে করে একটি Title তৈরি হবে)
                You are watching this sentence from collapsible...
                (এই লেখাটিই দেখাবে যখন collpasible arrow তে ক্লিক করা হবে)
                `<h1>Something</h1>`
                (html tag লেখার জন্য বেক টিক ` চিহ্ন ব্যবহার করা হয়)
            </details>

9. Table Style in markdown and github flavour

            table এর মধ্যে table head, table row & table data থাকে
            আর এসব তৈরি করা হয় যথাক্রমে... 
            Name | Class | Roll (এখানে এক একটি header/column আলাদা করার জন্য | চিহ্ন দেয়া হয়)
            ---- | ----- | ---- (এখানে একটা ডেস চিহ্ন দিলেই নিচে border তৈরি হয়ে যাবে)
            Sony | XII   | 79
            Boby | VIII  | 39

10. Image Style in markdown and github flavour

            image/logo সংযুক্ত করার পদ্ধতি যথাক্রমে... 
            ![logo](../logo/logo.png) 
            (প্রথমে আশ্চর্যবোধক চিহ্ন দিতে হবে তৃতীয় বন্ধনীর মধ্যে image/loge এর 
            title দিতে হবে, এরপর প্রথম বন্ধনীর মধ্যে image/logo path দিতে হবে)
            [![logo](../logo/logo.png)](https://google.com)
            (image/logo কে লিংকে পরিণত করতে এভাবে পুরো image/logo 
            কে তৃতীয় বন্ধনীর মধ্যে রেখে প্রথম বন্ধনীর মধ্যে link address দিতে হবে)
            <p align="center"><img src="../logo/log.png" width="100" title="logo"></p>
            (image/logo কে center align করতে একটি p tag এর মধ্যে নিতে হবে এবং p tag
            এর ভেতর align হিসেবে center উল্লেখ করে দিতে হবে।)

11. Codeblocks Style in markdown and github flavour

            কোনো কোডকে ব্লক আকারে দেখানো পদ্ধতি যথাক্রমে... 
            ``` (পর পর তিনটা বেক টিক চিহ্ন দিলেই লিখিত কোড ব্লক আকারে দেখাবে)
                <h1>Codeblocks</h1>
            ```
            ```html (পর পর তিনটি বেক টিকের পর html কথাটি উল্লেখ করে দিলে html এর tag color show করবে)
                <h1>github color flavour</h1>
            ```
            ```javascript (javascript কথাটি উল্লেখ করলে javascript এর tag color show করবে)
                console.log('github color flavour')
            ```

            <img src="https://img.shields.io/badge/idle%20teacher-it-brightgreen">
            Idleteacher It
            (https://shields.io এই website থেকে custom badge বানানোর

            web address copy করে img এর src তে বসিয়ে দিলেই কাঙ্ক্ষিত বেজ 
            দেখতে পাওয়া যাবে)

            <kbd>ctrl</kbd> + <kbd>c</kbd> ctrl + c

            Horizontal line by star

            <a name="headings"></a> (a tag এর ভেতরে name লিখে সেই id দিতে হবে যেটাকে কল করতে হবে)
            Headings
            <a name="emphasis"></a>
            Emphasis

            Escape অর্থাৎ কোনো কিছু পরিহার করার পদ্ধতি যথাক্রমে... 
            \- escape unorder list style 
            (ফরওয়ার্ড স্ল্যাশ পূর্বে দিলেই যেভাবে ডেস দিলে unorder list style দেখাত সেটার পরিবর্তে ডেস দেখাবে)
            Comment করার পদ্ধতি যথাক্রমে...
